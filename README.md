# LoopSession（プロトタイプ）

サーバーを経由せず、ユーザー同士が **P2P直接接続** でオンライン合奏するツールの試作。
リアルタイム性を追わず、**1小節ぶん意図的に遅延** させることでネットワーク遅延を実質的に無効化する。

> 発想の近い先行例に **NINJAM**（間隔同期ループでの遠隔ジャム）があります。
> 本プロジェクトの狙いは「サーバーレスP2P」「MIDI対応」「ルーパー的な複数モード」。

## 仕組み

- 全員が同じ **共有クロック**（BPM・拍子・小節番号）で動く。
- 手元で弾いた音（現在は MIDI／PCキーボード）は **すぐ自分に聞こえ**、相手には
  「小節N の位置X」という情報として送られ、相手側で **小節 N+`delayBars`** の同じ位置で再生される。
- 遅延要件は「ネットワーク往復 < 1小節の長さ」に緩和される
  （例: 120BPM・4/4 なら 1小節 = 2秒。ほぼどんな回線でも成立）。

### このプロトタイプの範囲

| 項目 | 現状 |
|---|---|
| データ | MIDI ノート（音声はまだ） |
| 接続 | インターネット P2P・手動シグナリング（オファー/アンサーをコピペ） |
| モード | 1小節ディレイ配信（弾き続けを1小節ずらす） |
| 人数 | 2人 |
| 音源 | Keys / Bass / Drums の3モード切替（各奏者が選択。自分=左パン・相手=右パン） |

### 楽器モード

「② 演奏」の音色ボタンで切り替え。選んだ楽器はノートに載せて送るので、相手側でも
同じ音色で（1小節遅れて）鳴る。掛け合いを聞き分けやすいよう自分=左・相手=右にパン。

- **🎹 Keys** — 三角波のシンセ（そのままの音程）
- **🎸 Bass** — ノコギリ波＋ローパス、2オクターブ下
- **🥁 Drums** — 合成ドラム。13キーに Kick/Snare/HH/OpenHH/Clap/Tom×3/Crash/Ride/Rim/Cow を割当

接続前でも音色は試聴できる（自分だけ鳴る）。役割分担の例: 片方が Drums、もう片方が Bass。

### ルーパー

「② 演奏」のルーパー行で操作。録音した音は毎ループ、通常演奏と同じ経路で
「自分は即モニター＋相手へは1小節遅れ」で送出されるため、**相手側の実装は不要**。

- **🎧 ソロ開始** — 相手がいなくてもメトロノーム＋ルーパーを動かす（一人での検証用）。
- **ループ長** — 1 / 2 / 4 小節。
- **● 録音** — 次のループ境界から録音開始 → ループ長ぶん記録 → 自動でループ再生。
  未接続なら自動でソロ開始。ループ再生中に再度押すと**オーバーダブ**（重ね録り）。
- **■ クリア** — ループを消去。

各音は**楽器情報ごと保存**されるので、ドラムを録ってループさせながら **Bass に持ち替えて
上で弾く**、といった検証ができる。

検証手順の例:
1. 🎧 ソロ開始（またはP2P接続）→ 🥁 Drums を選択
2. ● 録音 → メトロノームに合わせて1小節ぶん叩く → 自動でループ
3. 🎸 Bass に持ち替えてループに合わせて弾く（さらに ● 録音で重ねてもよい）

### オーディオ（音声転送・β）

オーディオI/Fの入力（ギター・ボーカル等）を **1小節遅れ**で相手へ送る Phase 2。

- 仕組み: 入力を小チャンク(Int16 PCM)に刻み、専用のバイナリ DataChannel で連続送信。
  相手側で **captureHost + delayBars×1小節** の位置に `AudioBufferSource` で並べて再生。
  整数小節ぶん遅らせるので拍が自動で揃う（MIDIと同じ思想）。
- キャプチャは **AudioWorklet**（`src/capture-worklet.js`）。エコー除去/AGC/ノイズ抑制は
  **オフ**（音楽用）、**モノラル**。
- **ヘッドホン必須級**（スピーカーだとハウリング）。**自分の音は機材側でモニター**
  （アプリ内モニターはレイテンシ回避のため無し）。
- 使い方: 接続（または🎧ソロ）→「③ オーディオ」で入力デバイスを選び **🎤 音声オン**。
  初回はブラウザのマイク許可が出る。レベルメーターが振れれば入力OK。
- 現状の制約: 生Int16 PCM（~384kbps/人）。将来 Opus 圧縮で削減可。音声のルーパーは未対応
  （まずはライブ送信）。ブラウザは Chrome/Edge 推奨。

## 動かし方（Node 不要）

このプロトタイプはビルド不要の素の JavaScript です。ローカルHTTPサーバーで開くだけ
（Web MIDI は `localhost` などの secure context が必要なため、`file://` 直開きは不可）。

```powershell
cd "C:\Users\xpyuu\OneDrive\デスクトップ\LoopSession"
python serve.py
```

> `python -m http.server` ではなく `serve.py` を使うこと。Windows の標準サーバーは
> `.js` を `text/plain` で返し、ブラウザが ES モジュールの実行を拒否するため。

ブラウザ（Chrome / Edge 推奨）で <http://localhost:5173> を開く。

### 2人で試す

一番手軽なのは **同じPCで2つのタブ** を開く方法（同一クロックなので同期はほぼ完璧）：

1. タブA（ホスト）: BPM等を決めて「1. オファーを作成」→ 生成された文字列をコピー。
2. タブB（参加者）: その文字列を「オファー」欄に貼り「2. アンサーを作成」→ 生成された文字列をコピー。
3. タブA: その文字列を「アンサー」欄に貼り「3. アンサーを取り込む → 接続」。
4. 接続後、どちらかで `A W S E D F ...` を叩くと、もう片方に **次の小節** で聞こえる。

別々のPC（別回線）でも同じ手順。オファー／アンサー文字列をチャット等でコピペ交換すればOK。

## 既知の制約・次の一手

- 手動シグナリングは 2人まで。多人数は接続手順が煩雑になる。
- クロック同期は往復対称を仮定した簡易版（RTT が非対称だとズレる）。
- 長時間で performance.now と AudioContext がドリフトし得る（開始時に再アンカーのみ）。

### ロードマップ

- **Phase 2**: 音声（録音）対応 — 1小節ぶんの PCM を Opus 圧縮して転送・スケジュール再生。
- **Phase 3**: ルーパー／オーバーダブ モード、多人数（メッシュ or 簡易シグナリング）、
  小節頭スナップの精度改善、TypeScript + Vite 化（Node 導入後）。
