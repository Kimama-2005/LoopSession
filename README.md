# LoopSession — P2P オンラインルーパー

サーバーレス P2P（WebRTC・手動シグナリング）でつながるオーディオルーパー。
縦長の DAW 風 UI。音の入り口は3つ:

1. **オーディオ I/F のライン入力**（ギター・マイク等）
2. **プラグイン音源 (WAM)** — アプリ内で Web Audio Modules のインストゥルメントを
   ホストし、MIDI キーボード / PC キーボードで演奏
3. **DAW のプラグイン音源** — DAW 出力をループバック/仮想デバイス経由で取り込み

公開版: https://kimama-2005.github.io/LoopSession/

## 仕組み

- リアルタイム音声は送らない。**録音が完了したループ (Int16 PCM) を DataChannel で転送**し、
  受信側が共有クロックのテンポグリッドに位相を合わせて再生する。
  ネットワーク遅延は「ループが1周遅れて届く」ことにしかならず、演奏はズレない。
- 共有クロックはオファー側の `performance.now()` が基準。アンサー側が
  ping/pong（NTP 方式・最小 RTT 採用）でオフセットを合わせる。
- 録音は次の小節頭から固定長（1/2/4/8小節）でクオンタイズ。
  オーバーダブはレイヤー追加方式（UNDO で1枚ずつ取り消し可能）。
- 入力は AEC/NS/AGC をすべて無効化して開く（ライン入力・音源入力を色付けしない）。
- 録音のレイテンシ補正: `baseLatency + outputLatency` を自動加算し、
  「設定 > 録音補正」で手動オフセットを足せる。

## 使い方

1. `python serve.py` → http://localhost:5173 （Chrome/Edge。`python -m http.server` は
   .js を text/plain で返すため不可）
2. START SESSION → 音の入り口を用意する
   - **ライン入力**: INPUT でオーディオ I/F のデバイスを選択
   - **プラグイン音源**: 「音源」で OBXD / TinySynth / Faust Flute（または任意の
     WAM2 プラグイン URL）を読み込む。MIDI キーボードは自動検出、
     KEYS を押すと PC キーボードでも演奏可（A W S E D... / Z X でオクターブ）。
     GUI ボタンでプラグインのパネルを開ける
   - **DAW の音源**: DAW の出力を I/F のループバックや VoiceMeeter 等の
     仮想デバイスへルーティングし、それを INPUT で選ぶ
3. トラックの ● を押すと次の小節頭から録音開始（停止中なら1小節カウントイン付きで自動スタート）。
   再生中に ● でオーバーダブ。
4. P2P 接続: 片方が「オファーを作る」→ コードを相手へ → 相手が貼って「受け取る」→
   返ってきた応答コードを最初の側が貼って「受け取る」。
   接続すると既存のループ・テンポ・走行状態も自動で同期される。

録音したループが遅れて聞こえる場合は「設定 > 録音補正」をプラス方向に調整。
モニターは基本 I/F 直（ダイレクトモニター）推奨。MON ボタンはブラウザ経由（遅延あり）。

## コード構成

- `src/engine.js` — 共有クロック・AudioContext・キャプチャバス・トランスポート・メトロノーム
- `src/capture-worklet.js` — 入力を約1024フレーム毎にフレーム番号付きで送る AudioWorklet
- `src/track.js` — ループスロット（固定長レイヤー録音・グリッド位相整列再生）
- `src/p2p.js` — 手動シグナリング WebRTC・クロック同期・ループ転送プロトコル・PCM 変換
- `src/instrument.js` — WAM2 プラグインホスト・WebMIDI・PC キーボード演奏
- `src/main.js` — UI とプロトコルの配線
- `legacy/` — 旧プロトタイプ（MIDI 合奏 + 音声ストリーミング β）

録音経路: ライン入力とプラグイン音源は `captureBus` に合流し、同じワークレットで
キャプチャされる（同時に鳴らして一緒に録ることも可能）。プラグインの読み込み先は
WAM 公式サンプルのホスティング（mainline.i3s.unice.fr）で、外部サーバーのため
落ちている場合は読み込みに失敗する（その場合もライン入力側は影響なし）。

## 既知の制限

- ピア間のオーディオクロックの個体差により、長時間（数十分）回し続けると
  数十 ms 単位のずれが蓄積しうる（トランスポート再スタートでリセット）。
- STUN のみ。対称 NAT 同士では繋がらない場合がある（必要になったら TURN を追加）。
- PCM 無圧縮転送（2小節ステレオ 48kHz ≒ 1MB 弱）。将来 Opus 化の余地あり。
- 2人接続（1対1）。
